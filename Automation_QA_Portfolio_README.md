# QA Portfolio - My Testing Approach

This portfolio demonstrates my professional approach as a QA Engineer.  
It shows how I analyse systems, design effective test coverage, build scalable automation frameworks, and ensure software quality through both manual and automated testing.

My work focuses on **test strategy, automation architecture, exploratory testing, and maintainable test design.**

---

## 1. TransferMarkt Test Automation Framework

**Repository:**  
👉 https://github.com/yanamak89/TransferMarktTestFramework

**Description:**  
A test automation framework for the TransferMarkt website built using **C# .NET and Playwright**.

The framework automates key user flows including navigation, Premier League table validation, search behaviour, login scenarios, and selected API interactions.

The project demonstrates modern automation engineering practices including:

- Page Object Model (POM)
- reusable test utilities
- structured test organisation
- maintainable selectors
- separation of concerns

The framework architecture follows **SOLID, DRY, and KISS principles**, making the solution scalable and easy to extend.

**Tech stack:**

- C#
- .NET
- Playwright
- NUnit
